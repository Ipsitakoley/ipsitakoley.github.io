Project 3: Stability Analysis under Different Ethernet Gateway Strategies

Objective

Due to recent growth in feature rich cyber physical systems like automobiles, the concept of shared Electrical Control Units (ECU) has come into the picture. Instead of having different ECUs for different control components, different controllers can be mapped to same ECU. Based on the periodicity, deadlines and disturbance margins of each control task, mapped to same ECU, it can decide on certain control strategies to accommodate them. One of them is dropping certain control executions in order to accommodate some other control task within the residual bandwidth.[Refer 5]

  1. ECU1 is connected to ECU2 and camera via Ethernet Gateway. Camera sends image information through this gateway. ECU2 has ADAS Application which processes the camera data and sends feature extracted data to ECU1.
  2. ECU2 is part of CAN network and it can be assumed some other ECUs are also connected to the same gateway causing some traffic. [Refer 2 & 4]
  3. Different strategies used in gateway affects the network traffic and delays the arrival of feature data to ECU1.
  4. We decide if it’s a deadline miss based on calculated periodicity of the task in ECU1 and Response Time of the network, i.e. tn+rt>t(n+1) where tn=time at nth period, t(n+1)=time at (n+1)th period and rt=response time of the network.
  5. Based on the deadline miss we can generate a pattern when deadline miss contributes a ‘0’ and execution within deadline contributes ‘1’.
  6. Given a recognizer for all supported patterns to run the target plant following some performance metric, it needs to be figured out if the current pattern is recognizable by it.
  7. If not, the network must be altered so that a recognizable pattern can be generated. [Refer 1]
